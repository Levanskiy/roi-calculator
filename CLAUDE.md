# ROI Calculator (Dubai) — схема проекта

Одностраничный калькулятор доходности недвижимости в Дубае. Статический сайт без сборки —
один `index.html` со встроенным CSS/JS. Хостится на GitHub Pages для доступа с телефона
и компьютера по постоянной ссылке.

## Структура

```
roi-calculator/
├── index.html              ← всё приложение (UI + расчёты + генерация PDF-отчёта)
├── manifest.webmanifest     ← PWA-манифест (установка на главный экран)
├── sw.js                     ← service worker (офлайн-кэш)
├── icons/                    ← иконки PWA (192/512/maskable)
└── CLAUDE.md                 ← вы здесь
```

## Логика расчёта (index.html)

- **Сценарии**: готовый объект / офф-план · наличные / ипотека · долгосрочная / краткосрочная аренда — переключаются через `M = {prop, fin, rent}` и `setMode()`. Ипотека недоступна для офф-плана (всегда наличные).
- `calc()` — все формулы (DLD, комиссии, ипотечный платёж, NOI, cash-on-cash, ROI за 5 лет). Для офф-плана возвращает дополнительный объект `offplan` с таймлайн-расчётами.
- `updatePlanCheck()` — пересчитывает и подсвечивает контрольную строку плана платежей (см. ниже); вызывается из `recalc()` и `syncVisibility()`.
- `offplanTimeline(r)` — общая функция, строит периоды Стройка → Handover → [Post-handover] → Далее (используется и на экране, и в PDF).
- `recalc()` — рендерит блок результатов при каждом изменении инпута.
- `buildReport()` / `downloadPDF()` — экран отчёта и экспорт в PDF (html2canvas + jsPDF, подключены с CDN).

### Офф-план: модель плана платежей

- Вводимые поля: «До handover, %» (`paidPct`) и «На handover, %» (`handoverPct`) — это реальные платежи (во время стройки и на передаче ключей), которые есть почти всегда, независимо от того, включён ли PHP.
- Без PHP (`phpEnabled` не отмечен): `paidPct + handoverPct` должны давать 100% — контрольная строка показывает «Итого по плану, %» и подсвечивается ошибкой, если сумма ≠ 100%.
- С PHP (`phpEnabled` отмечен, поле `phpMonths` становится видимым): «Post-handover, %» (`phpPct`) — остаток `100% − paidPct − handoverPct` (computed, не вводится), показан в той же контрольной строке. Если остаток < 0 — подсвечивается ошибкой (для post-handover не остаётся места).
- До handover: ни аренды, ни операционных расходов. Рост стоимости — `growthPre` (%/год). На handover актив переоценивается: `valueAtHandover = P × (1+growthPre/100)^(месяцы/12)`.
- После handover рост идёт по `growthPost`. Если включён PHP — ежемесячный платёж застройщику `P × phpPct/100 / phpMonths`, чистый поток = NOI − этот платёж («Аренда покрывает рассрочку на X%»).
- Горизонт 5 лет учитывает таймлайн: аренда считается только за годы после handover, а в годы PHP — за вычетом платежей застройщику.

## Изменение допущений по умолчанию

Стартовые значения полей (цена, площадь, ставки и т.д.) — это атрибуты `value="..."` у
`<input>` в `index.html`. DLD/комиссии/трасти-сборы — константы внутри `calc()`.

## Деплой (GitHub Pages)

1. Закоммитить изменения в `main`.
2. `git push` — Pages пересобирается автоматически (Settings → Pages → Deploy from branch `main` / root).
3. Ссылка остаётся той же: https://levanskiy.github.io/roi-calculator/

После любых правок в `index.html`/иконках — обновить `CACHE` версию в `sw.js` (например `roi-dubai-v2`), иначе у пользователей со старым кэшем не подхватится новая версия сразу.

## Локальная проверка

```bash
cd roi-calculator
python3 -m http.server 8000
# открыть http://localhost:8000
```

(Service worker не зарегистрируется на `file://`, нужен http-сервер.)
